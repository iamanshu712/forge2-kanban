# Forge 2 · Kanban — Reference Build

A tiny Trello-style Kanban board: **Laravel API (SQLite) + React (Vite) UI**.
Boards → Lists → Cards, with tags, member assignment, and due dates.

> **Honest note on provenance:** the Forge 2 qualifier is graded specifically on
> whether *your OpenClaw + Hermes agents*, driven by you through Slack, wrote this
> code — that's what `agent-log.md` and the Slack export are for. The code in this
> repo was scaffolded directly (not through an agent chat loop), so **it's a
> reference/starting point for you to learn from and re-drive through your own
> agent setup — not a drop-in qualifier submission.** Swap in your own
> `agent-log.md`, Slack export, and `evidence/walkthrough.mp4` showing your agents
> actually doing the work before submitting anything.

## What's here

- `/backend` — Laravel 11 API, SQLite, models + migrations + controllers for
  Boards, Lists, Cards, Tags, Members.
- `/frontend` — React + Vite UI: board picker, drag-and-drop lists/cards, card
  detail modal (description, due date, member, tags).
- `/skills/status-report/SKILL.md` — the Hermes skill the handbook requires.

## Required features implemented

- ✅ Boards → Lists → Cards, create/edit/delete at every level
- ✅ Move a card between lists (drag-and-drop, calls `PATCH /cards/{id}/move`)
- ✅ Card title + description, editable
- ✅ Coloured tags on cards (create, attach, detach)
- ✅ Board members + assign a member to a card
- ✅ Due dates, with overdue cards visually flagged (red left border + "⚠ overdue")

## Run it locally

### Backend (Laravel)

This repo ships the **application code** (models, migrations, controllers,
routes) but not the Laravel framework scaffold itself (vendor/, artisan, etc.) —
that has to be generated by Composer, which needs network access.

```bash
# 1. Scaffold a fresh Laravel app, then copy our app code over it
composer create-project laravel/laravel backend-scaffold
cp -r backend/app/. backend-scaffold/app/
cp -r backend/database/migrations/. backend-scaffold/database/migrations/
cp backend/routes/api.php backend-scaffold/routes/api.php
cd backend-scaffold
cp .env.example .env   # then copy values from backend/.env.example on top
php artisan key:generate

# 2. SQLite setup
touch database/database.sqlite
# make sure .env has: DB_CONNECTION=sqlite

# 3. Migrate + serve
php artisan migrate
php artisan serve   # http://localhost:8000
```

### Frontend (React + Vite)

```bash
cd frontend
npm install
cp .env.example .env   # VITE_API_URL=http://localhost:8000/api
npm run dev            # http://localhost:5173
```

Open `http://localhost:5173`, create a board, and go — the app auto-creates
"To Do / Doing / Done" lists for every new board.
